How this pipeline works

UCC filings → consolidation accounts → MCA leads — operator guide

Understanding the numbers

A full Florida run might show about 2,451 filings, ~1,946 accounts, and ~169 MCA lead accounts (businesses with MCA funders on filings), with roughly ~12 priority and ~5 work in the consolidation queue. Filing-level leads rows from strict detection may still be only ~8—that is normal.

Filings
Every UCC row we scraped from FL—banks, SBA, registered agents, MCA funders, and more. Stored in ucc_filings. This number is always the largest.
Leads (MCA funder accounts)
One business_accounts row per debtor with ≥1 MCA funder on linked filings (lender_class=mca_funder or is_mca). Manage names in MCA lenders and reclassify after edits.
Accounts
One business per normalized debtor name + state in business_accounts. consolidation_score and consolidation_tier (priority / work / monitor / suppress) are your main call list for debt consolidation.

Why two “lead” numbers? The dashboard total is MCA funder accounts. The legacy leads table counts strict per-filing detect_mca matches only.

Work Leads for MCA funder businesses. Use Accounts for the full consolidation tier queue. Browse Filings for research.

Core concepts

UCC filing
A public lien record from a Secretary of State. Each row in ucc_filings is one filing (debtor, secured party, dates).
MCA lead
A business account with at least one MCA-classified secured party on its filings. Lender names come from mca_aliases (classifier) plus keyword heuristics.
Consolidation account
One business (normalized debtor name + state) rolled up across its filings in business_accounts. Scored for debt-consolidation outreach — separate from MCA lead scoring.

Workflow (Florida today)

  1. Manual FL scrape — Deep pagination on the Florida API (row numbers, multiple index profiles). Optional filing-date enrichment per row.
  2. Persist filings — Upsert by state + filing_number (no duplicates).
  3. Classify & rollup — Tag lender class / MCA on filings; merge into business_accounts.
  4. Score accounts — consolidation_score (0–100) and consolidation_tier: priority, work, monitor, suppress.
  5. MCA pipeline — Creates or updates leads with MCA match metadata.

Florida Standard search ignores A–Z text cursors; coverage comes from paginating rowNumber across org compact + org actual index profiles (and optional Exact lender terms).

How to run

  • Dashboard — On /dashboard/, use Run Florida scraper (POST /dashboard/scrapers/FL/run, background job).
  • CLI — Fresh data: python scripts/reset_fl_data.py then python scripts/run_fl_scrape.py. Quick test: add --quick. Snapshot: python scripts/report_fl_data.py.

Scores & tiers

Field Where Meaning
consolidation_score business_accounts 0–100 debt-consolidation fit from filing patterns (funder count, recency, material UCCs).
consolidation_tier business_accounts priority / work / monitor / suppress — who to call first.
lead_score leads MCA match strength (alias / fuzzy / heuristic), not consolidation fit.

Operator checklist

  • Before a full refresh: reset_fl_data.py (FL only).
  • Run scrape; confirm new rows on Filings and Accounts.
  • Do not worry if leads << filings on a broad scrape—that is expected.
  • Sort accounts by tier — review priority and work first (your consolidation queue).
  • Check leads for MCA names; seed mca_aliases if hit rate is low.
  • Investigate fuzzy debtor matches flagged “needs review” on account detail.
